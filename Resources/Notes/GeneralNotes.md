# General Notes - Digital Nurture 5.0

## Key Takeaways
- **Design Patterns**: Used to solve recurring software design problems. Creating loose coupling using interfaces is key.
- **DSA**: Emphasizes time and space complexity. Optimal algorithms prevent performance bottlenecks.
- **PL/SQL**: Relational database operations running server-side. Triggers, procedures, and functions improve security and transaction integrity.
- **TDD**: Write tests before writing logic. JUnit 5 and Mockito facilitate decoupled, verifiable code development.

## Best Practices
1. Follow standard Java naming conventions (camelCase for methods, PascalCase for classes).
2. Document edge cases in PL/SQL triggers (like handling NULL values or transaction rollbacks).
3. Ensure unit tests are independent and follow the AAA (Arrange, Act, Assert) pattern.
